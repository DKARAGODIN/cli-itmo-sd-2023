# Последовательность работы

## Запуск, интерфейс

Запускаем приложение. Сначала создается объект класса `Application`. Затем у созданного объекта запускается метод `run`.

### Application

У объекта данного класса есть: поле `contextManager`, метод `run`.

В `contextManager` сохранются все переменные окружения.Метод `run` запускает бесконечный цикл, внутри которого последовательно выполняются следующие операции:

1. Принять ввод до символа переноса строки.

2. Если строка пустая, то переходим к пункту 1, иначе происходит обработка входной строки.

3. Разбиение входной строки с помощью `Lexer`. Если `Lexer` успешно обработал строку то передаем результат в `Substituter`, иначе выводим ошибку и переходим к пункту 1.

4. Подстановка переменных вместе лексем с помощью `Substituter`.

5. Построение `Executable` с помощью `Parser`.

6. Последовательное исполнение `Executable` с передачей вывода предыдущих команд последующим при наличии пайпов.

7. В случае успешного завершения всех `Executable` печатается вывод. В случае возникновения ошибки во время исполнения одного из `Executable` выполнение прекращается и печается информации об ошибке.

8. В случае выполнения команды `exit` работа приложения завершается.
